# 소프트웨어 개발 프로젝트 업무 지시서

## 개요
이 문서는 소프트웨어 개발 프로젝트에서의 일반적인 업무 지시서로, 프로젝트 진행 중 발생할 수 있는 문제를 예방하고 효율적인 개발을 지원하기 위한 가이드라인입니다. 특히 모듈화 작업, 데이터 처리, 기능 구현 등에서의 best practice를 담고 있습니다.

## 1. 프로젝트 구조 설계

### 1.1 폴더 구조
```
project_name/
├── src/                 # 소스 코드
│   ├── config/          # 설정 파일
│   ├── models/          # 데이터 모델
│   ├── modules/         # 기능 모듈
│   ├── routes/          # 라우터 (Flask/Django 등)
│   ├── services/        # 비즈니스 로직
│   └── utils/           # 유틸리티 함수
├── tests/               # 테스트 코드
├── docs/                # 문서
├── data/                # 데이터 파일
├── configs/             # 설정 파일 (JSON/YAML)
└── README.md            # 프로젝트 설명
```

### 1.2 모듈화 원칙
1. **단일 책임 원칙**: 각 모듈은 하나의 기능만 담당
2. **개방-폐쇄 원칙**: 확장에는 열려 있고, 변경에는 닫혀 있어야 함
3. **의존 역전 원칙**: 고수준 모듈은 저수준 모듈에 의존하지 않아야 함
4. **인터페이스 분리 원칙**: 큰 인터페이스보다 작은 인터페이스를 선호

### 2.2 함수/메서드 구현
```python
def calculate_total_price(items):
    """상품 총 가격 계산"""
    total = 0
    for item in items:
        total += item['price'] * item['quantity']
    return total
```
- 함수는 하나의 기능만 수행
- docstring으로 함수 설명 작성
- 예외 처리 구현

### 3.3 예: 비속어 필터 기능 구현
```python
class ProfanityFilter:
    """비속어 필터 클래스"""
    
    def __init__(self, config_path):
        """초기화"""
        self.config = self._load_config(config_path)
        self.profanity_words = self.config.get("profanity_words", [])
    
    def filter_text(self, text, remove_profanity=True):
        """텍스트 필터링"""
        if remove_profanity:
            for word in self.profanity_words:
                if word in text:
                    text = text.replace(word, "***")
        return text
```
