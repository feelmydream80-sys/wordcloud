# 워드클라우드 감정 분석 시스템 - FP 분석 보고서

## 문서 정보
- **프로젝트명**: 워드클라우드 감정 분석 시스템
- **작성일**: 2026년 1월 19일
- **분석 방법**: IFPUG Function Point Analysis 4.3.1

- **예상 노력**: 8-12 인월 (Python 기반 AI 프로젝트 기준)

## 1. 프로젝트 개요

본 프로젝트는 텍스트 데이터를 입력받아 NLP 분석, 감정 분석, 워드클라우드 생성을 수행하는 AI 기반 시스템입니다. KoCoSa와 KOTE 데이터셋을 활용한 모델 파인튜닝과 웹 기반 사용자 인터페이스를 포함합니다.

### 1.1 주요 기능
- 텍스트 형태소 분석 (Kiwi, Okt)
- 감정 분석 (3-class, 44-class 다중 레이블)
- 풍자 분석 (2-class)
- 워드클라우드 생성 및 시각화
- 모델 파인튜닝 (KoCoSa, KOTE 데이터셋)
- 웹 기반 실시간 분석
- 모델 성능 비교 및 평가

## 2. FP 산정 방법론

### 2.1 FP 구성 요소
- **EI (External Input)**: 시스템 외부에서 입력되는 데이터/명령
- **EO (External Output)**: 시스템 외부로 출력되는 데이터
- **EQ (External Inquiry)**: 시스템 외부의 조회 요청에 대한 응답
- **ILF (Internal Logical File)**: 시스템 내부에서 유지되는 논리적 데이터
- **EIF (External Interface File)**: 외부 시스템과의 인터페이스 파일

### 2.2 복잡도 평가 기준
- **Low**: 단순한 데이터 구조, 기본적인 처리 로직
- **Average**: 중간 정도의 복잡성
- **High**: 복잡한 데이터 구조, 고급 처리 로직

## 3. 기능별 FP 상세 분석

### 3.1 데이터 처리 및 분석 모듈 (152 FP)

#### 3.1.1 NLP 분석 모듈 (45 FP)
**기능 설명**: 텍스트를 입력받아 형태소 분석을 수행하고 결과를 JSON 형식으로 출력. CLI에서는 'inputs/sample_text.txt' 파일을 읽어 분석하며, 웹에서는 '/analyze' 엔드포인트로 실시간 텍스트를 처리.

**FP 분류 및 근거**:
- **EI (External Input)**: 텍스트 입력 (DET: 3, FTR: 2) → Low 복잡도 → 3 FP
  - DET (Data Element Types): 텍스트 내용, 분석 옵션, 출력 경로
  - FTR (File Type Referenced): 설정 파일 참조
  - **기능 사례**: CLI 실행 시 'inputs/sample_text.txt' 파일 읽기, 웹에서 다중 문장 텍스트 입력 (예: "영화가 정말 재미있었어. 날씨가 너무 추워서 짜증 나.")
- **EO (External Output)**: 분석 결과 출력 (DET: 8, FTR: 1) → Average 복잡도 → 4 FP
  - DET: Kiwi 토큰, Okt 형태소, 품사 태그, 의미 단어 등
  - **기능 사례**: JSON 파일로 Kiwi 토큰 리스트 (['영화', '가', '정말', '재미있', '었', '어']), Okt 형태소 분석 결과 (품사 태그 포함), 의미 단어 리스트 (['영화', '재미있']) 출력
- **ILF (Internal Logical File)**: 분석 설정 (DET: 6, RET: 2) → Low 복잡도 → 5 FP
  - DET: 분석기 옵션, 품사 태그 설정, 출력 형식 등
  - **기능 사례**: 'configs/nlp_config.json' 파일에 Kiwi/Okt 분석기 설정, 워드클라우드 품사 필터 옵션 (["Noun", "Verb", "Adjective"]) 저장

**소계**: 3 + 4 + 5 = 12 FP (기본) × 3.75 (복잡도 가중치) = 45 FP
