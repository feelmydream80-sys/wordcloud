## 분석 배경 / 이유
- SSE 완료 신호가 UI에 전달되지 않는 split-brain 버그 발생
- 모든 직원이 실패하는 NLP 설정 파일 경로 오류
- StopwordManager/NLPnalysis 동시 초기화 시 JVM 중복 시작 문제
- Flask SSE + POST 동시 처리 불가 문제

---

## 분석 결과

### 1. Split-brain 버그 (P0)
- 원인: `batch_service.py`에서 `batch_processing_state = {...}` 재할당으로 새 dict 객체 생성
- SSE `stream_batch_events()`는 이전 dict 객체 참조 → UI에 "파일 로드 중..." 영구 표시
- 수정: in-place mutation (`batch_processing_state['key'] = ...`)으로 통일

### 2. NLP Config 경로 오류 (P0, 근본 원인)
- 원인: `NLPnalysis.get_instance()` 기본 인자 `"configs/nlp_config.json"`가 상대 경로
- Flask CWD가 `web/`이므로 파일 미존재 → `FileNotFoundError` → Stage 2 전체 실패
- 수정: `settings.py`에서 절대 경로 `NLP_CONFIG_PATH`, `CONFIGS_DIR_PATH` 계산
- `batch_processor.py`에서 상대 경로 대신 절대 경로 사용

### 3. StopwordManager 동시성
- 원인: `ThreadPoolExecutor` 여러 worker가 동시에 `get_stopword_manager()` 호출
- 수정: `_stopword_manager_lock = threading.Lock()` + Double-Checked Locking 패턴 적용

### 4. Column Mapping 저장/로드
- 원인: `load_last_mappings()`에서 0바이트 `last_mapping.json` → `JSONDecodeError` → 500 응답
- 수정: 빈 파일/손상 파일 처리, `save_mappings()`에서 빈 dict 거부
- 보완: 0바이트 파일 삭제

### 5. Batch_processor 기타 수정
| 항목 | 원인 | 수정 |
|------|------|------|
| WordCloudGenerator 공유 | 여러 worker가 동일 인스턴스 사용 → 경합 | 각 worker가 자체 생성 |
| batch_id 오류 | `create_batch_summary`가 잘못된 ID 계산 | `os.path.basename(batch_dir)` 사용 |
| word*freq TypeError | `str * float` 연산 불가 | `str(word) * max(1, int(freq))` |

### 6. JS Frontend 수정
- `columnMappings` 초기화 누락 → 새 파일 업로드 시 NaN
- `retryFailed()` 파라미터 누락 → retry 시 mapping/컬러 옵션 누락
- `deleteBatch()` 경로 추측 로직 → `sessionStorage` 직접 사용
- XSS: `escapeHtml()` 적용 (employee_id, error, profanities)
- SSE: `JSON.parse()` try/catch, 빈 데이터 체크
- mapping 저장: `/api/mappings/save` 응답 체크
